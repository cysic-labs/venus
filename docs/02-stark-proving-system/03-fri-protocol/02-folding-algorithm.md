# FRI Folding Algorithm

## Overview

The folding algorithm is the core mechanism that drives FRI's efficiency, enabling logarithmic verification of polynomial degree bounds. Each folding step transforms a polynomial of degree d into a polynomial of degree d/2 using a random challenge from the verifier. This degree halving continues until reaching a small constant, at which point the polynomial can be verified directly.

Folding exploits the algebraic structure of polynomials over finite fields. Any polynomial can be uniquely decomposed into even and odd components, and a random linear combination of these components produces a lower-degree polynomial while preserving the essential property: if the original was genuinely low-degree, so is the folded result; if not, the inconsistency becomes detectable.

This document provides a detailed treatment of the folding algorithm, its mathematical basis, implementation considerations, and optimizations used in practice.

## Mathematical Foundation

### Polynomial Decomposition

Every polynomial P(X) of degree less than 2n can be uniquely written as:

```
P(X) = P_even(X^2) + X * P_odd(X^2)
```

where P_even and P_odd are polynomials of degree less than n.

If P(X) = a_0 + a_1*X + a_2*X^2 + a_3*X^3 + ... + a_{2n-1}*X^{2n-1}, then:

```
P_even(Y) = a_0 + a_2*Y + a_4*Y^2 + ... + a_{2n-2}*Y^{n-1}
P_odd(Y) = a_1 + a_3*Y + a_5*Y^2 + ... + a_{2n-1}*Y^{n-1}
```

### The Folding Operation

Given a random field element alpha provided by the verifier, the folded polynomial is:

```
P'(Y) = P_even(Y) + alpha * P_odd(Y)
```

Key properties:
- deg(P') < deg(P)/2 when P has the claimed degree
- P' is defined over Y = X^2, effectively a domain of half the size
- The combination with alpha is crucial for soundness

### Relationship Between Evaluations

For any point x with y = x^2, the folded value can be computed from P(x) and P(-x):

```
P(x) = P_even(y) + x * P_odd(y)
P(-x) = P_even(y) - x * P_odd(y)
```

Solving these equations:

```
P_even(y) = (P(x) + P(-x)) / 2
P_odd(y) = (P(x) - P(-x)) / (2x)
```

Therefore:

```
P'(y) = P_even(y) + alpha * P_odd(y)
      = (P(x) + P(-x))/2 + alpha * (P(x) - P(-x))/(2x)
```

This formula allows computing folded values directly from evaluations without knowing coefficients.

### Evaluation Form Computation

In FRI, polynomials are typically represented by their evaluations rather than coefficients. The folding operation in evaluation form:

```
Given: Evaluations of P at domain D = {d_0, d_1, ..., d_{n-1}}
       where d_i and -d_i are paired (d_i^2 = d_{i+n/2}^2 = y_i)

Compute: For each i from 0 to n/2 - 1:
         P'(y_i) = (P(d_i) + P(-d_i))/2 + alpha * (P(d_i) - P(-d_i))/(2*d_i)
```

The folded domain D' = {y_0, y_1, ..., y_{n/2-1}} = {d_0^2, d_1^2, ...}.

## Domain Structure for Folding

### Multiplicative Group Requirements

For efficient folding, the evaluation domain must have specific structure:

```
Domain D must satisfy:
1. |D| = 2^k for some k (power of two size)
2. D forms a multiplicative group under field multiplication
3. For each x in D, -x is also in D
4. The set D' = {x^2 : x in D} has size |D|/2
```

### Primitive Roots of Unity

The standard choice uses roots of unity:

```
Let omega be a primitive n-th root of unity (omega^n = 1, omega^j != 1 for 0 < j < n)

Domain D = {omega^0, omega^1, omega^2, ..., omega^{n-1}}

Property: omega^{n/2} = -1, so omega^{i+n/2} = -omega^i

Folded domain D' = {omega^{2*0}, omega^{2*1}, ..., omega^{2*(n/2-1)}}
                 = domain generated by omega^2, which is primitive (n/2)-th root
```

### Pairing Structure

Elements pair naturally for folding:

```
Pair (omega^i, omega^{i+n/2}) satisfies:
- omega^i + omega^{i+n/2} = omega^i - omega^i = 0 (sum is zero)
- omega^i * omega^{i+n/2} = omega^i * (-omega^i) = -omega^{2i} (product is negative square)
- (omega^i)^2 = (omega^{i+n/2})^2 = omega^{2i} (squares are equal)
```

### Coset Domains

When avoiding the trace domain, use cosets:

```
Coset: g * D = {g*omega^0, g*omega^1, ..., g*omega^{n-1}}

where g is a generator not in D (g is not an n-th root of unity)

Folding still works:
- (g*omega^i)^2 = g^2 * omega^{2i}
- Folded domain is g^2 * D' (also a coset)
```

## FRI Folding Protocol

### Single Folding Round

One round of FRI folding:

```
Input: Commitment to P of degree < d on domain D
       Random challenge alpha

Prover:
1. Compute P_even(Y), P_odd(Y) from P(X)
2. Compute P'(Y) = P_even(Y) + alpha * P_odd(Y)
3. Evaluate P' on folded domain D'
4. Build Merkle tree over P' evaluations
5. Send Merkle root as commitment to P'

Output: Commitment to P' of degree < d/2 on domain D'
```

### Complete FRI Commit Phase

The full commit phase iterates folding:

```
Initialize:
  P_0 = original polynomial
  D_0 = initial domain
  d_0 = degree bound

For round i = 0, 1, 2, ... until d_i is small:
  1. Prover commits to P_i (Merkle root r_i)
  2. Verifier sends challenge alpha_i = Hash(r_0 || r_1 || ... || r_i)
  3. Prover computes P_{i+1} by folding P_i with alpha_i
  4. D_{i+1} = folded domain, d_{i+1} = d_i / 2

Final:
  Prover sends P_final coefficients directly (small polynomial)
```

### Number of Rounds

The number of folding rounds depends on:

```
Initial degree bound: d_0
Final degree bound: d_final (typically small constant, e.g., 1, 2, or 4)

Number of rounds: log_2(d_0 / d_final)

Example: d_0 = 2^20, d_final = 4
         Rounds = log_2(2^20 / 4) = log_2(2^18) = 18
```

## Implementation Strategies

### Coefficient-Based Folding

When polynomial is in coefficient form:

```
Input: Coefficients [a_0, a_1, a_2, ..., a_{2n-1}]
       Challenge alpha

Process:
  Even coefficients: [a_0, a_2, a_4, ..., a_{2n-2}]
  Odd coefficients:  [a_1, a_3, a_5, ..., a_{2n-1}]

  Folded[i] = a_{2i} + alpha * a_{2i+1}

Output: Coefficients [a_0 + alpha*a_1, a_2 + alpha*a_3, ..., a_{2n-2} + alpha*a_{2n-1}]
```

This is simple but requires coefficient representation.

### Evaluation-Based Folding

When polynomial is in evaluation form (more common):

```
Input: Evaluations [P(d_0), P(d_1), ..., P(d_{n-1})] on domain D
       Challenge alpha
       Domain D with pairing structure

Process:
  For i = 0 to n/2 - 1:
    x = d_i
    minus_x = d_{i + n/2}  // = -x
    y = x^2

    p_even = (P(x) + P(minus_x)) / 2
    p_odd = (P(x) - P(minus_x)) / (2 * x)

    P'(y) = p_even + alpha * p_odd

Output: Evaluations [P'(y_0), P'(y_1), ..., P'(y_{n/2-1})]
```

### Optimized Evaluation Folding

Avoid divisions by precomputing:

```
Precompute: two_inv = 2^{-1} in the field
            x_inv[i] = (2 * d_i)^{-1} for all i

Folding:
  For i = 0 to n/2 - 1:
    sum = P(d_i) + P(d_{i + n/2})
    diff = P(d_i) - P(d_{i + n/2})

    P'(y_i) = sum * two_inv + alpha * diff * x_inv[i]
```

Batch inversions can compute all x_inv values efficiently.

### Restructured Formula

An alternative formulation avoids separate even/odd computation:

```
P'(y) = (1 + alpha/x) * P(x) / 2 + (1 - alpha/x) * P(-x) / 2

Let beta = alpha * x^{-1}
P'(y) = ((1 + beta) * P(x) + (1 - beta) * P(-x)) / 2
```

This form may be more efficient for certain hardware.

## Batched Folding

### Multiple Polynomials

When multiple polynomials need FRI proof, combine with random weights:

```
Polynomials: P_0, P_1, P_2, ..., P_k
Batching challenge: beta (from verifier, before folding starts)

Combined: P = P_0 + beta*P_1 + beta^2*P_2 + ... + beta^k*P_k

Apply FRI to combined polynomial P
```

### Quotient Polynomial Batching

For STARK, typically batch:
- Trace polynomial quotients
- Composition polynomial quotients
- DEEP polynomial components

```
Q_combined = Q_0 + gamma*Q_1 + gamma^2*Q_2 + ... + gamma^m*Q_m

where gamma is a random challenge
```

### Evaluation Combination

At each evaluation point, combine:

```
P_combined(x) = P_0(x) + beta*P_1(x) + beta^2*P_2(x) + ...
```

This allows single Merkle tree for multiple polynomials.

## Memory and Streaming

### In-Place Folding

Memory-efficient in-place algorithm:

```
Evaluations stored in array A[0..n-1]
Paired elements at A[i] and A[i + n/2]

For i = 0 to n/2 - 1:
  Compute folded value from A[i] and A[i + n/2]
  Store in A[i]  // Reuse first half of array

Result: Folded evaluations in A[0..n/2-1]
        Second half A[n/2..n-1] no longer needed
```

### Streaming Folding

For very large polynomials that don't fit in memory:

```
1. Read pairs (A[i], A[i+n/2]) from disk
2. Compute folded values
3. Write to new file
4. Build Merkle tree in streaming fashion
```

This enables proving polynomials larger than available RAM.

### Multi-Level Folding

Fold multiple levels at once to reduce I/O:

```
Instead of: Fold once, commit, get challenge, fold again, ...

Do: Get challenges alpha_0, alpha_1 upfront
    Fold twice in one pass (degree d -> d/4)
    Commit to intermediate result

Trade-off: Requires more challenges upfront (affects transcript structure)
```

## Parallelization

### Independent Pairs

Folding parallelizes naturally:

```
For i = 0 to n/2 - 1 in parallel:
  Compute P'(y_i) from P(x_i), P(-x_i)
```

Each pair is independent; no synchronization needed within a round.

### GPU Implementation

Efficient GPU folding:

```
1. Load evaluation pairs to GPU memory
2. Precompute inverses on GPU (batch inverse algorithm)
3. Launch kernel with one thread per pair
4. Each thread computes folded value
5. Copy results or build Merkle tree on GPU
```

### Memory Bandwidth Optimization

For large polynomials, memory bandwidth dominates:

```
Optimize access patterns:
- Coalesce reads: A[i] and A[i+n/2] in same cache line if possible
- Streaming stores: Write results sequentially
- Prefetching: Load next batch while computing current
```

## Correctness Verification

### Verifier's Check

During FRI query, verifier checks folding correctness:

```
Given: Query position x in original domain
       Claimed values P(x) and P(-x)
       Claimed folded value P'(y) where y = x^2
       Challenge alpha

Verify:
  p_even = (P(x) + P(-x)) / 2
  p_odd = (P(x) - P(-x)) / (2 * x)
  expected = p_even + alpha * p_odd

  Check: expected == P'(y)
```

### Error Detection

If prover's folded polynomial is incorrect:

```
True folded value: P'_correct(y)
Prover's claimed value: P'_claimed(y)

At random query x:
- If P'_correct(y) != P'_claimed(y), check fails
- Probability of matching at random point: 1/|F| (negligible)
```

### Multi-Round Verification

Verifier checks all folding rounds:

```
For each query position x_0:
  y_0 = x_0^2
  y_1 = y_0^2 = x_0^4
  ...

  Check: P_0(x_0), P_0(-x_0) fold to P_1(y_0)
  Check: P_1(y_0), P_1(-y_0) fold to P_2(y_1)
  ... (through all layers)

  Check: Final layer matches claimed constant
```

## Key Concepts

- **Folding**: Combining even and odd polynomial parts to halve degree
- **Domain pairing**: Structure where x and -x share squared value y = x^2
- **Evaluation folding**: Computing folded values from paired evaluations
- **Batching**: Combining multiple polynomials with random weights
- **In-place**: Memory-efficient folding reusing evaluation array

## Design Considerations

### Implementation Choices

| Aspect | Option A | Option B |
|--------|----------|----------|
| Representation | Coefficient | Evaluation |
| Memory | In-place | Copy |
| Parallelism | CPU threads | GPU |
| I/O | Memory-resident | Streaming |

### Numerical Precision

In finite fields, no floating-point issues. However:
- Use Montgomery multiplication for efficiency
- Batch inversions to reduce expensive operations
- Avoid redundant reductions modulo p
